Survey API (Go)
Objetivo

Desenvolva uma API REST em Go para receber respostas de um formulário de pesquisa científica e armazená-las em um banco PostgreSQL.

A API deve ser pequena, simples, segura, bem organizada e pronta para produção, seguindo as melhores práticas do ecossistema Go.

O foco é simplicidade, legibilidade e facilidade de manutenção. Evite qualquer tipo de overengineering.

Stack

Utilize obrigatoriamente:

Go 1.24+
PostgreSQL
pgx/v5
chi (router)
Docker
Docker Compose
golang-migrate
godotenv (ou configuração equivalente)
context.Context em todas as operações

Não utilize frameworks pesados.

Banco

Criar apenas uma tabela.

Campos:

Handler

Service

O Service deve conter toda a regra de negócio.

Exemplo:

validar dados

Repository

Responsável apenas pelo acesso ao PostgreSQL.

Não colocar regra de negócio.

Utilizar sempre:

ctx context.Context
Context

Todas as operações devem receber context.Context.

Exemplo:

func (s *Service) Create(ctx context.Context, survey Survey) error

Nunca utilizar context.Background() dentro das camadas da aplicação.

O contexto deve ser propagado desde o Handler até o Repository.

Dependências

Utilizar injeção de dependências por construtores.

Exemplo:

service := survey.NewService(repository)

handler := survey.NewHandler(service)

Evitar singletons e variáveis globais.

Objetivo final

Quero um projeto que sirva como template pessoal para APIs simples em Go. O código deve parecer escrito por um desenvolvedor Go experiente, privilegiando simplicidade, clareza, organização e facilidade de manutenção.

A configuração deve exigir apenas alterar o arquivo .env, executar docker compose up -d e utilizar a rota POST /send-form. Nenhuma configuração adicional deve ser necessária.

Importante: antes de gerar qualquer código, faça um planejamento da arquitetura, explique brevemente as decisões (como uma mini ADR) e só então implemente o projeto completo, arquivo por arquivo, seguindo exatamente a estrutura proposta.
